# Getting started

## Install

`tsr` is a single static binary. Build it from source with Cargo:

```sh
cargo build --release       # → target/release/tsr
```

Then put `target/release/tsr` on your `PATH`.

## Scaffold a config

From your repo root, generate a starter `tasks.toml`:

```sh
tsr --init
```

This writes a commented `tasks.toml` showcasing all three task forms. It refuses
to overwrite an existing file, so it is always safe to run.

<Callout type="info" title="tasks.toml is the workspace anchor">
  The location of `tasks.toml` defines your workspace root. `tsr` finds it by
  walking up from the current directory to the nearest `tasks.toml`, so you can
  run tasks from anywhere in the repo.
</Callout>

## Define a task

```toml
[tasks.dev]
run = "vite"
dir = "apps/web"
```

Run it:

```sh
tsr dev
```

Because `run = "vite"` has no shell metacharacters, `tsr` splits it and spawns
`vite` directly — no `npm run` / Node startup tax.

## Forward arguments

Everything after `--` is forwarded to the resolved command:

```sh
tsr test -- --watch        # → <resolved test command> --watch
```

## List what's defined

```sh
tsr list
```

```text
Available tasks:
  build  delegate: turbo
  dev    run: vite  ·  dir: apps/web
  test   packages: apps/*
```
